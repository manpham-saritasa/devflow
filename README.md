# Devflow

Automatically copy AI workflow files from devflow to other projects whenever devflow updates.

## What does it do?

Instead of manually copy-pasting `.ai/` files and AI rules between projects, run one command to sync everything. You choose which AI tools and workflow files you want. After devflow updates, just run one command again to get the latest versions.

This tool is currently in development and not yet published to npm.

---

## Part 1: Using it locally (during development)

### Requirements

- Node.js 18 or higher
- Git

### Setup (one-time, on your machine)

Follow these steps once to set up devflow-sync on your computer.

**Step 1:** Open terminal, navigate to the devflow repository folder
```bash
cd /path/to/devflow/cli
```

**Step 2:** Install dependencies
```bash
npm install
```

**Step 3:** Build the tool
```bash
npm run build
```

**Step 4:** Make the command available globally
```bash
npm link
```

After this, you can use `devflow-sync` command anywhere on your machine.

### How to use in another project

Once setup is complete, go to any project you want to sync.

**Step 5:** Open terminal, go to your project folder
```bash
cd /path/to/your-project-folder
```

**Step 6 (first time):** Run init to choose what to sync
```bash
devflow-sync init
```

This asks you questions:
- Which AI tools does your project use? (Claude, Copilot, etc.)
- Which components do you want? (agents, skills, prompts, instructions, startup)
- For each component: sync all items or pick specific ones?
- How to handle files that already exist? (overwrite, backup, or skip)

A `.devflow.json` file is created — keep this file in your project.

**Step 7 (after devflow updates):** Re-sync automatically
```bash
devflow-sync update
```

This re-syncs using your previous choices — no questions asked.

**Optional: Preview first**
```bash
devflow-sync update --dry-run
```

Shows what would change without writing any files.

### What gets synced?

| Component | What it is |
|-----------|-----------|
| **agents** | AI workflow agents (planner, coder, reviewer, etc.) |
| **skills** | Automation workflows for specific tasks |
| **prompts** | Reusable prompts and templates |
| **instructions** | Coding rules and communication guidelines |
| **startup** | Session entry point configuration |

### The .devflow.json file

After `devflow-sync init`, a file `.devflow.json` appears in your project. This file remembers your choices:
- Which AI tools you use
- Which components to sync
- Which specific items within each component

Do not edit this file manually. If you want to change your choices, run `devflow-sync init` again.

## Part 2: After it's published to npm

When devflow-sync is published to npm, setup becomes simpler.

### How to use (published version)

**First time in a project:**
```bash
npx devflow-sync init
```

**After devflow updates:**
```bash
npx devflow-sync update
```

**Preview changes without writing:**
```bash
npx devflow-sync update --dry-run
```

The interactive prompts and workflow are the same. No local setup needed — just run the commands.

## Questions?

If something doesn't work:

1. Check that Node.js 18+ is installed: `node --version`
2. Rebuild the tool: `npm run build` (from the cli folder)
3. Make sure you ran `npm link` after building
4. Run `devflow-sync init` instead of `update` to reconfigure from scratch
